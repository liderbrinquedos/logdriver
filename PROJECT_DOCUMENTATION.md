# Log Lider Project Documentation

## Overview
Log Lider is a web application designed for delivery management, specifically tailored for motorists (drivers). The system allows tracking of deliveries from creation to confirmation, including document uploads (canhotos) and status management.

## Technology Stack
- **Backend**: FastAPI (Python 3.12)
- **Database**: SQLite
- **Frontend**: HTML/JavaScript (with PWA capabilities via service worker)
- **Server**: Nginx (reverse proxy)
- **API Documentation**: Auto-generated by FastAPI (Swagger UI available at `/docs`)

## Database Schema

### Deliveries Table
The core entity in the system is the `deliveries` table, which stores all delivery-related information.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | Unique identifier for each delivery |
| nf_number | VARCHAR | NOT NULL, INDEXED | Nota Fiscal number (delivery identifier) |
| client | VARCHAR | NOT NULL | Client name |
| address | VARCHAR | NOT NULL | Delivery address |
| city | VARCHAR | NULLABLE | City of delivery |
| phone | VARCHAR | NULLABLE | Contact phone number |
| value | FLOAT | NULLABLE | Delivery value/amount |
| driver | VARCHAR | NULLABLE | Driver's name responsible for delivery |
| company | VARCHAR | NULLABLE | Company associated (AGR, APL, LDR) |
| observations | TEXT | NULLABLE | General observations about the delivery |
| canhoto_path | VARCHAR | NULLABLE | File path to the original canhoto (document) image |
| status | VARCHAR | DEFAULT 'pendente', INDEXED | Current status: pendente, entregue, cancelado |
| delivery_canhoto_path | VARCHAR | NULLABLE | File path to the delivery confirmation canhoto image |
| delivery_observations | TEXT | NULLABLE | Observations made at the time of delivery confirmation |
| delivered_at | DATETIME | NULLABLE | Timestamp when delivery was confirmed as completed |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | Timestamp when delivery record was created |

### Indexes
- Primary key on `id`
- Index on `nf_number` for faster searches by Nota Fiscal
- Index on `status` for filtering deliveries by status

## Business Rules

### Delivery Statuses
A delivery can have one of three statuses:
1. **pendente** (pending) - Initial state when delivery is created
2. **entregue** (delivered) - When delivery has been confirmed as completed
3. **cancelado** (cancelled) - When delivery is cancelled

### Status Transition Rules
- From **pendente** → can transition to **entregue** (via confirmation) or **cancelado**
- From **entregue** → can transition back to **pendente** (which clears delivery-specific data)
- From **cancelado** → no further transitions allowed (terminal state)

### Data Validation Rules
#### Delivery Creation
- `nf_number`: Required, minimum 1 character
- `client`: Required, minimum 1 character
- `address`: Required, minimum 1 character
- `city`: Optional
- `phone`: Optional
- `value`: Optional, must be a valid float if provided
- `driver`: Optional
- `company`: Optional
- `observations`: Optional
- `canhoto`: Optional file upload (image)

#### Status Update
- `status`: Must be one of: 'pendente', 'entregue', 'cancelado'

#### Delivery Confirmation
- `delivery_observations`: Optional text
- `delivery_canhoto`: Optional file upload (image)

### File Upload Rules
- Uploaded files are stored in the `/uploads` directory
- Filename format: `{timestamp}_NF{nf_number}_{suffix}{extension}`
  - `timestamp`: Unix timestamp
  - `nf_number`: Delivery's Nota Fiscal number
  - `suffix`: Either "original" (for initial canhoto) or "entrega" (for delivery confirmation)
  - `extension`: Original file extension (defaults to .jpg if none provided)
- Files are accessible via URL path: `/uploads/{filename}`

### Special Business Logic
1. When changing status from 'entregue' back to 'pendente':
   - Delivery confirmation canhoto path is cleared
   - Delivery observations are cleared
   - Delivery timestamp is cleared

2. When confirming a delivery (setting status to 'entregue'):
   - Delivery confirmation canhoto is saved if provided
   - Delivery observations are recorded
   - Delivery timestamp is set to current time

3. Search functionality:
   - Supports searching by client name, nf_number, or address (case-insensitive partial match)
   - Can filter by status (pendente, entregue, cancelado, or all)

4. Statistics:
   - Total deliveries
   - Delivered deliveries (status = 'entregue')
   - Pending deliveries (status = 'pendente')
   - Cancelled deliveries (status = 'cancelado')

## API Endpoints

### Health Check
- **GET** `/api/health`
  - Returns: `{"status": "ok", "timestamp": "ISO 8601 timestamp"}`

### Statistics
- **GET** `/api/stats`
  - Returns: DeliveryStats object with total, entregues, pendentes, cancelados counts

### Deliveries Collection
- **GET** `/api/deliveries`
  - Query Parameters:
    - `search` (string, optional): Search term for client, nf_number, or address
    - `status` (string, optional): Filter by status (pendente, entregue, cancelado, or omit for all)
    - `skip` (integer, default 0): Number of records to skip for pagination
    - `limit` (integer, default 100): Maximum number of records to return
  - Returns: Array of DeliveryResponse objects

- **POST** `/api/deliveries`
  - Form Parameters:
    - `nf_number` (string, required)
    - `client` (string, required)
    - `address` (string, required)
    - `city` (string, optional)
    - `phone` (string, optional)
    - `value` (float, optional)
    - `driver` (string, optional)
    - `company` (string, optional)
    - `observations` (string, optional)
    - `canhoto` (file, optional): Upload file for original canhoto
  - Returns: DeliveryResponse object of created delivery
  - Status Code: 201 Created

### Single Delivery Operations
- **GET** `/api/deliveries/{delivery_id}`
  - Path Parameter: `delivery_id` (integer)
  - Returns: DeliveryResponse object

- **PATCH** `/api/deliveries/{delivery_id}/status`
  - Path Parameter: `delivery_id` (integer)
  - Request Body: DeliveryStatusUpdate object (status: string, must be valid status)
  - Returns: DeliveryResponse object
  - Business Rule: Changing from 'entregue' to 'pendente' clears delivery confirmation data

- **PATCH** `/api/deliveries/{delivery_id}/confirm`
  - Path Parameter: `delivery_id` (integer)
  - Form Parameters:
    - `delivery_observations` (string, optional)
    - `delivery_canhoto` (file, optional): Upload file for delivery confirmation canhoto
  - Returns: DeliveryResponse object
  - Effects: Sets status to 'entregue', saves delivery confirmation data, sets delivered_at timestamp

- **DELETE** `/api/deliveries/{delivery_id}`
  - Path Parameter: `delivery_id` (integer)
  - Returns: No content (204 No Content)

## Security
- CORS middleware configured to allow all origins (for development)
- In production, this should be restricted to specific domains
- No authentication implemented in current version (would need to be added for production use)

## File Storage
- Upload directory: `./uploads/` (relative to backend directory)
- Served as static files at `/uploads` URL path
- Files are not automatically cleaned up when deliveries are deleted (consider implementing cleanup)

## Deployment Notes
- The application uses Uvicorn as the ASGI server
- Nginx configuration is provided for reverse proxy setup
- Service worker (`sw.js`) provides PWA capabilities for offline use and caching
- Environment variables are not currently used; database URL is hardcoded (consider using environment variables for configuration)
